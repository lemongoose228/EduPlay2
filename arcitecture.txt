Runtime interaction
-------------------
1) Frontend works with backend via REST (axios) for CRUD and session actions.
2) Frontend connects to `/sessions` namespace via Socket.IO for live quiz events.
3) Backend `SessionsGateway` validates WS JWT and broadcasts state changes.
4) Quiz timers are stored in Redis keys with TTL.
5) On key expiration, `SessionsTimerListener` triggers reveal/transition logic and emits updates to clients.
6) Local-only sessions (own, crocodile, wheel, station, tictactoe) run on the host screen; join-by-code is blocked for these types.
7) Quiz is the only multiplayer session type (invite code + WebSocket sync).
8) Tic-tac-toe uses REST endpoints (`/tictactoe/setup`, `/open-cell`, `/answer`), not WebSocket.
9) `SessionsCleanupService` periodically deletes sessions older than `SESSION_RETENTION_DAYS` (default 90).
