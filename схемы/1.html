Модель данных · ER-диаграмма

Ключевые сущности: пользователи, игры, категории, вопросы, сессии, команды, игроки, лайки и жалобы.

            USERS {
              uuid id PK
              string email UK
              string name
              string role
              boolean isBlocked
              string avatar
            }

            GAMES {
              uuid id PK
              uuid authorId FK
              string title
              string type
              string status
              jsonb settings
            }

            CATEGORIES {
              uuid id PK
              uuid gameId FK
              string name
              int order
            }

            QUESTIONS {
              uuid id PK
              uuid categoryId FK
              int value
              text question
              text answer
            }

            GAME_LIKES {
              uuid userId PK
              uuid gameId PK
            }

            SESSIONS {
              uuid id PK
              string inviteCode UK
              uuid gameId FK
              uuid hostId FK
              string status
              jsonb settings
            }

            TEAMS {
              uuid id PK
              uuid sessionId FK
              string name
              int score
            }

            PLAYERS {
              uuid id PK
              uuid teamId FK
              string name
              string userId
              boolean isHost
            }

            REPORTS {
              uuid id PK
              uuid gameId FK
              uuid reporterUserId FK
              string status
              uuid reviewedByUserId FK
              text reason
            }
Game имеет поля type (своя игра/викторина/крокодил/колесо), статус модерации и JSON-настройки. В сессиях хранится состояние игр.

Варианты использования · акторы и сценарии

Гость / Пользователь / Создатель / Администратор. Ключевые расширения и включения: создание сессий, библиотека, модерация.
