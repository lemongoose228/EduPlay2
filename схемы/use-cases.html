Варианты использования

Акторы и сценарии с учётом библиотеки, сессий, редакторов типов игр, профиля и модерации. Связи «include» / «extend» отражают типичные зависимости сценариев.

Создатель по смыслу расширяет пользователя (дополнительные UC); администратор — отдельная роль в системе (user / admin / super_admin). При необходимости нарисуйте обобщение отдельной стрелкой в UML-редакторе. 

usecaseDiagram
  actor Guest as Гость
  actor User as Пользователь
  actor Creator as Создатель
  actor Admin as Администратор

  (Регистрация и вход) as UC_auth
  (Поиск игр) as UC_search
  (Публичная библиотека) as UC_public
  (Избранное) as UC_fav
  (Присоединение по коду) as UC_join
  (Создать сессию) as UC_create_sess
  (Вести сессию ведущим) as UC_host
  (Играть в команде) as UC_play
  (Пауза и завершение) as UC_lifecycle
  (Настройки профиля) as UC_profile
  (Жалоба на игру) as UC_report
  (Личные игры) as UC_my
  (Редактор из шаблона) as UC_builder
  (Выбор типа игры в редакторе) as UC_types
  (Сохранить черновик) as UC_draft
  (Опубликовать игру) as UC_publish
  (Очередь жалоб) as UC_reports
  (Блокировка контента) as UC_block
